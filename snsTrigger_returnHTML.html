What is this?

This is a tool which allows you to see what impact a feature switch is having on an individual page's performance. This is done through a custom implementation of the testing tool Lighthouse running in an AWS Lambda instance.

What exactly does it test and why?

When you provide a URL & Feature Switch, first the tool runs tests against 3 different versions of the provided URL to establish a performance baseline.
The 3 different versions are:

  • default - Most closely matches what users see when coming to the site, but is the most unpredictable do to the variations in the size of ad content & speed of 3rd party services outside of kinja's control.
  • adzone=kinjatest - Similar to the default version, but removes the variances caused by advertisements by always forcing our static test ads to be used.
  • no3rdparty - Removes all potential variations by only permitting things that kinja has direct control over to load. Prevents things like calls to 3rd-party services responsible for analytics, ad verification, header bidding, etc.

It then runs tests against the same 3 different versions of the URL, but this time appends the feature switch to each test.

Once all 6 of the tests are completed, the tool matches up each version of the test with it's feature switched counter-part version and compares the results of the meterics:

  • Time to First Meaningful Paint - How long it takes to before the primary content is painted on the page
  • Total Byte Weight - Total size of the page after everything is loaded
  • DOM Nodes - Total number of nodes (individual parts) that exist on the page. This is used as a proxy to gauge the complexity of the page
Example Test

URL: https://gizmodo.com/

feature switch: freepizza=on

The different URLs tested:

  • https://gizmodo.com/
  • https://gizmodo.com/?freepizza=on
  • https://gizmodo.com/?adzone=kinjatest
  • https://gizmodo.com/?adzone=kinjatest&freepizza=on
  • https://gizmodo.com/?no3rdparty
  • https://gizmodo.com/?no3rdparty&freepizza=on

The blue, red, & green groups are then compared.

Other Questions
Can I test more then one feature switch at a time?
Sort of.
The test runner will test whatever you provide in the "Feature Switch" field against the baseline values of the 3 different variations. That means in the example above, if you provide the "feature switch" value of freepizza=on&icecream=chocolate then URLs that would be tested would be:
https://gizmodo.com/
https://gizmodo.com/?freepizza=on&icecream=chocolate
https://gizmodo.com/?adzone=kinjatest&
https://gizmodo.com/?adzone=kinjatest&freepizza=on&icecream=chocolate
https://gizmodo.com/?no3rdparty
https://gizmodo.com/?no3rdparty&freepizza=on&icecream=chocolate
Why are some of the I clicked submit and my tests aren't there?
When the page loads, it grabs all of the completed tests from the database, attempts to perform the matching & comparison analysis, and then displays the results on the page.
If you have just submitted a test, they generally take anywhere from 30 - 180 seconds to complete before they will be available in the database and if your tests aren't in the database prior to page loading, they will not be displayed.

Make sure that you have provided the correct "URL to Test". It's best to copy & paste directly from another browser window. Things like using http instead of https or adding or forgetting www can potential cause issues with running your tests.

If after making sure your fields are correct, having waited some time, and refreshing the page you tests are still not there, reach out to someone on the "Performance" team.
Why don't tests show more information?
The 3 metrics that are displayed have been picked as a simple way of tracking and understanding performance on our sites. That being said, all of the performance audit data for each individual URL test is available in JSON form. Click on the ⎘ icon under "Copy result JSON to Clipboard", and then paste the text into a editor or a beautifier tool to make it easier to read.

Additionally, it is worth noting that Chrome has the Lighthouse tool built into it's included dev tools. This means that you can run tests against individual URLs using your browser. It won't do the work of comparing different URLs, but it does provide significantly more information that our version excludes.
What do the different icons mean?
⚖ - Indicates a comparison test and is created when a value is provided in the "Feature Switch" field.
🎯 - Indicates a singular test and is created when a value is not provided in the "Feature Switch" field.
⚠ - Indicates that a issue has occurred with a test. This is most commonly seen when only half of a comparison test has been completed.
